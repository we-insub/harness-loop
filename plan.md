# 하네스&루프 블로그 생성 도구 개발 계획

## 1. 목표

`/Users/matowe/Desktop/하네스루프` 디렉토리 안에서 CLI 또는 AI 에이전트를 이용해 네이버 블로그용 정보성 글을 생성한다.

사용자는 제목, 원문 데이터, 이미지 태그, API Key 또는 에이전트 로그인 정보를 입력하고, 필요하면 체크박스로 루프 실행을 켠 뒤 N회 반복 생성 결과 중 가장 규칙을 잘 지킨 결과물을 선택한다.

최종 결과물은 지정된 출력 형식과 절대 규칙을 만족해야 한다.

## 2. 핵심 요구사항

### 2.1 입력 방식

- 사용자가 제목을 입력한다.
- 사용자가 원문 데이터를 입력한다.
- 원문에는 `[image_1.jpg]`, `[image_2.jpg]` 같은 이미지 태그가 포함될 수 있다.
- API Key를 직접 입력하거나 `.env`에서 불러올 수 있게 한다.
- AI 에이전트 로그인 방식도 확장 가능하도록 구조를 열어둔다.
- 루프 실행 여부를 체크박스로 선택한다.
- 루프 실행 체크 시 반복 횟수 N을 입력받는다.

### 2.2 생성 결과

- 제목은 사용자가 입력한 원문 제목 그대로 유지한다.
- 본문은 `제목을입력해주세요1:`과 `본문2:` 형식을 지킨다.
- 본문 도입부에는 방문일자를 명시한다.
- 방문일자가 원문에 없으면 `2026-05-17 ~ 2026-06-15` 사이 임의 날짜를 사용한다.
- 정보성 데이터가 있으면 해당 소제목 안에 표를 반드시 생성한다.
- 이미지 태그는 순서와 위치를 유지한다.
- 소제목은 `ㅂㅂㅂ소제목명` 형식만 사용한다.
- 목차에는 `ㅂㅂㅂ` 마커를 붙이지 않는다.
- 마크다운 볼드, HTML 표, 마크다운 표를 사용하지 않는다.

## 3. 프롬프트 영역 분리

긴 프롬프트를 하나로 관리하지 않고, 아래 영역별 파일로 나눠 관리한다.

```text
prompts/
├─ role.txt
├─ date_rules.txt
├─ hard_rules.txt
├─ style_rules.txt
├─ mobile_rules.txt
├─ image_rules.txt
├─ table_rules.txt
├─ subtitle_rules.txt
└─ output_format.txt
```

### 3.1 role.txt

- N 블로그 상위노출 전문 에디터 역할 정의
- 차분한 한국어 정보성 글 작성 목표 정의

### 3.2 date_rules.txt

- 오늘 날짜: 2026-06-16
- 방문일자 자동 생성 범위: 2026-05-17 ~ 2026-06-15
- 범위 밖 날짜 금지
- 도입부에 `YYYY.MM.DD 방문 기준` 형식 명시

### 3.3 hard_rules.txt

- 제목 100% 그대로 유지
- 정보성 소제목에는 표 필수
- 원문 데이터 누락 금지
- 정보 70 : 경험 30 비율 준수

### 3.4 style_rules.txt

- 차분하고 진정성 있는 에세이 톤
- 자연스러운 어미 사용
- AI 특유 표현 금지
- 과도한 감성 표현 제한
- 마크다운 기호 금지

### 3.5 mobile_rules.txt

- 한 줄 10~15자 내외
- 의미 단위 줄바꿈
- 마침표 뒤 빈 줄 확보
- 긴 문단 금지

### 3.6 image_rules.txt

- 원문 이미지 태그 유지
- 순서와 위치 변경 금지
- 한 줄에 이미지 태그 하나만 출력
- 이미지 태그 바로 아래 개별 캡션 금지

### 3.7 table_rules.txt

- 표 마커 형식 강제
- HTML 표 금지
- 마크다운 표 금지
- 셀 좌표 형식 검증 가능하게 작성

### 3.8 subtitle_rules.txt

- 본문 소제목은 `ㅂㅂㅂ소제목명`
- 소제목 최소 3개 이상
- 목차에는 `ㅂㅂㅂ` 금지

### 3.9 output_format.txt

- 최종 출력 형식 고정
- `제목을입력해주세요1:`
- `본문2:`

## 4. 하네스 실행 흐름

기본 실행 흐름은 다음과 같다.

```text
입력값 수집
→ 프롬프트 조립
→ AI 생성
→ 규칙 검증
→ 실패 항목 리포트
→ 필요 시 자동 수정
→ 최종 결과 저장
```

## 5. 루프 기능

### 5.1 루프 비활성화

루프 체크박스가 꺼져 있으면 아래 흐름으로 실행한다.

```text
1회 생성
→ 1회 검증
→ 필요 시 1회 수정
→ 최종 저장
```

### 5.2 루프 활성화

루프 체크박스가 켜져 있으면 N회 반복 실행한다.

```text
N회 생성
→ 각 결과 검증
→ 점수화
→ 가장 통과율 높은 결과 선택
→ 필요 시 최종 수정
→ best.txt 저장
```

### 5.3 루프 결과 저장

```text
outputs/
├─ run_001.txt
├─ run_002.txt
├─ run_003.txt
├─ best.txt
└─ report.json
```

## 6. 검증기 규칙

검증기는 생성 결과를 문자열 기준으로 검사한다.

### 6.1 필수 검증 항목

- 제목이 입력 제목과 완전히 일치하는지 확인
- `제목을입력해주세요1:` 존재 확인
- `본문2:` 존재 확인
- 소제목이 최소 3개 이상인지 확인
- 본문 소제목이 모두 `ㅂㅂㅂ`로 시작하는지 확인
- 목차 항목에 `ㅂㅂㅂ`가 들어가지 않았는지 확인
- 원문 이미지 태그가 모두 존재하는지 확인
- 이미지 태그 순서가 원문과 같은지 확인
- 방문일자가 `2026-05-17 ~ 2026-06-15` 범위 안인지 확인
- 정보성 데이터가 있는 경우 표가 1개 이상 있는지 확인
- 표 블록이 `표 N x M 시작`과 `표 N x M 끝` 형식을 지키는지 확인
- 마크다운 표 사용 여부 확인
- HTML 표 사용 여부 확인
- `**` 볼드 마커 사용 여부 확인
- 금지 표현 사용 여부 확인

### 6.2 점수화 기준

각 검증 항목은 통과 또는 실패로 기록한다.

루프 실행 시 통과 항목 수가 가장 많은 결과를 우선 선택한다.

동점일 경우 제목, 이미지, 날짜, 표 규칙을 더 많이 통과한 결과를 우선한다.

## 10. 완료 기준

아래 조건을 만족하면 1차 버전을 완료로 본다.

- 사용자가 제목과 원문을 입력할 수 있다.
- API Key를 이용해 AI 생성이 가능하다.
- 루프 체크박스를 켜고 N회 반복 생성할 수 있다.
- 각 결과가 `outputs/run_XXX.txt`로 저장된다.
- 가장 규칙을 잘 지킨 결과가 `outputs/best.txt`로 저장된다.
- 검증 결과가 `outputs/report.json`으로 저장된다.
- 제목, 이미지 태그, 방문일자, 소제목, 표 형식 검증이 동작한다.
